Full Stack react app

Occasions

An app that lets you set up special occasions so you don't forget to send a card to your friends and family.

I used Firebase for the backend with React-Redux-Firebase for the state managment. NodeJs and Firebase Cloud Functions to set up back end logic.

Some functionality:
- Profile creator with firebase authentication and imput validation done in the front end.
- Create occasions, choose a card and add it to the cart to schedule it to be sent at a later date.
- Back end logic with google scheduler to set up a daily cron job that checks if any card needs to be sent today.
- Filter card information to be used in an html template and sent it with nodemailer.

Medieval Personal Shopper

I heard a good project to have in your portfolio is an e-commerce site so to make it fun for myself, I made it a game based on the rpgs I grew up loving.

Diferent heroes come in with a set ammount of money to spend and an enemy to fight.You have to buy items for them to beat their stats.

For the fights I created a small engine with javascript classes and recursive functions that ends up working like an autobattler. You can even use spells!

The main two objectives were to learn React class components with life cycle hooks and React Redux for content managment. It also has a call to an api with redux thunk for data stored in contentful.

ZuckerMash

A React project that parodies FaceMash, Mark Zuckerberg's app made before Facebook. Instead of rating college women on their looks, you rate pictures of Mark.

It is made with Create React App, functional components and react hooks, React Router for pagination and React Context api for state managment.

Some funtionality includes:
- An engine to randomize images based on certain criteria like tags and previous images displayed.
- A voting system and a ranking page to display the most voted pictures.
- A memory system to go back to previous images.

Random Curse Generator

A generator that randomly picks a funny curse for the user.

It was made with vanilla javascript, html, sass for css with the gsap framework for the animations.

To get the curse, there's a small mechanic of extinguishing a set of flames by clicking on them. A small function checks a counter on each flame and if they are all at cero, an animation is triggered.

The cursed are displayed randomly from an array and the resulting one is saved in the local storage. You cannont change your curse (unless you clear local storage).
